# spec-sync — 漂移兜底（非首选路径）

> ⚠️ **首选不是这个。** 最佳实践是开发/拷问时**内联**更新 `CONTEXT.md` 和 ADR（用 `/grill-with-docs`），让文档**永不漂移**。
> 本技能是**兜底**：当你已经写了代码、文档落后了，用它一次性追平。能内联就别等到这一步。

## 步骤

### 1. 找出改了什么
看最近改动（git diff，或你描述的本次工作）。识别：新术语？接口/数据结构变了？模块增删？做了架构决策？

### 2. 对照更新

| 改动 | 更新目标 |
|------|---------|
| 新术语 | `CONTEXT.md`（按 `toolkit/CONTEXT.template.md` 格式，含 `_Avoid_`；只放项目特有术语，不放实现细节） |
| 接口变更 | 对应 `docs/modules/XXX.md`（若存在） |
| 架构决策 | 触发 `/adr`（先过"难逆+反直觉+真权衡"三条） |
| 任务完成 | `docs/PROJECT_STATUS.md`（若存在） |

### 3. 检测漂移
发现代码与文档**矛盾**（如文档说 bcrypt、代码用 md5）→ **停下报告**，不擅自抹平，让用户定夺哪个是对的。

### 4. 报告
列出更新了哪些文件/行；漂移单独高亮。

## 原则
- 用完这次之后，下次试着改用 `/grill-with-docs` 内联更新，逐步减少对兜底的依赖。
