## 🤖 AI 协作指南

### 文档体系（按需阅读，不要全读）

| 文档 | 何时读 |
|------|--------|
| `CONTEXT.md` | **每次开发前** — 查领域术语，统一命名 |
| `docs/adr/` | 想知道"为什么这样设计"时 |
| `docs/modules/XXX.md` | 改动该复杂/敏感模块前（如果存在） |

### 工作流程

- **普通功能** → 读 `CONTEXT.md`，用一致命名快速实现
- **复杂/敏感模块** → 严格按 `docs/modules/XXX.md` 实现
- **遇到新术语** → 用 `/grill-me` 盘问并更新 `CONTEXT.md`
- **做了架构决策** → 用 `/adr` 记录
- **改完代码** → 用 `/spec-sync` 同步文档

### 何时为模块写详细文档？

满足任一条件才写 `docs/modules/XXX.md`：
- 涉及安全（认证/授权/加密）
- 处理支付或敏感数据
- 多步骤复杂业务逻辑
- AI 反复理解错

其余模块只在 `CONTEXT.md` 定义术语即可，保持灵活。
