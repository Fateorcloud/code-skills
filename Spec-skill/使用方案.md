# 使用方案（精简完整版）

一页讲清:**有哪些工具、怎么用、什么时候用哪个**。

> 内核致敬 [mattpocock/skills](https://github.com/mattpocock/skills):`grill-me` / `grill-with-docs` 的拷问法、`CONTEXT.md`(有主见的术语表)+ `docs/adr/` 结构、ADR 三铁律,全部吸收自他。本项目做的是**中文化 + 学习化封装 + 生命周期补全**(`spec-init`/`spec-check`),不声称比他更优,只服务他没覆盖的中文学习场景。

## 🧰 工具清单

### 6 个技能(任意项目里输入 `/技能名` 触发)

| 技能 | 干什么 | 什么时候用 | 来源 |
|------|--------|-----------|------|
| `/spec-init` | 提问 + 生成 CLAUDE.md、CONTEXT.md、首个 ADR | **新项目第一步** | 本项目原创 |
| `/grill-me` | 就方案反复拷问你(一次一问、每问给建议),**不写文档** | 想把设计想透 / "grill me" | 吸收自 mattpocock |
| `/grill-with-docs` | 拷问 + **内联**把术语写进 CONTEXT.md、决策记成 ADR | 边想边沉淀(**首选**) | 吸收自 mattpocock |
| `/adr` | 记一条架构决策(三铁律才记,默认 1-3 句) | 做了难逆的技术选型 | 格式吸收自 mattpocock |
| `/spec-sync` | 文档漂移后**兜底**追平 | 没内联、文档落后了 | 本项目原创 |
| `/spec-check` | 只读体检,找矛盾/死链/术语表不纯净 | 定期 / 接手老项目 | 本项目原创 |

技能源码在 `.claude/skills/`(项目级,仅本项目可用),可直接编辑。

### 4 个模板 + 1 个脚本(`toolkit/`,复制即用)

- `CONTEXT.template.md` — 有主见的术语表(每词带 `_Avoid_`)
- `ADR.template.md` — 极简架构决策(1-3 句 + 三铁律)
- `CLAUDE.template.md` / `MODULE.template.md`
- `spec-init.sh` — 一键把目录+模板+技能拷进新项目

```bash

/grill-with-docs                                  # 2. 边拷问边沉淀术语/决策

```

## 🔁 日常循环

```
要开发功能
  │
  ├ 想把方案/术语想透 ──→ /grill-with-docs(边问边内联更新 CONTEXT.md、必要时记 ADR)
  │                       纯想方案不碰文档 ──→ /grill-me
  │
  ├ 功能复杂 or 安全敏感(认证/支付/隐私)?
  │     ├ 是 → 写 docs/modules/XXX.md(套 MODULE.template),让 AI"严格按此实现"
  │     └ 否 → 让 AI"按 CONTEXT.md 的术语实现",灵活快速
  │
  ├ 做了难逆+反直觉+真权衡的决策? ──→ /adr
  │
  └ 忘了内联、文档落后了 ──→ /spec-sync 兜底;定期 /spec-check 体检
```

## 📐 五条铁律(全部来自吸收 mattpocock 的实践)

1. **CONTEXT.md 只是术语表** —— 只放项目特有术语,绝不放实现细节、不放通用编程概念。
2. **术语要有主见** —— 一个概念多个词,选最佳的,其余进 `_Avoid_`,全项目统一用词。
3. **ADR 默认 1-3 句** —— 三条同时成立才记:难逆 + 反直觉 + 真权衡;否则别记。
4. **内联更新 > 事后同步** —— 拷问/开发时当场更文档(`/grill-with-docs`),`spec-sync` 只是兜底。
5. **从 3 个文件起步,懒创建** —— 只给复杂/敏感模块写详细 spec。

## 📂 最终项目长这样

```
your-project/
├── CLAUDE.md              # AI 入口
├── CONTEXT.md             # 有主见的术语表(单上下文)
│   └─ 或 CONTEXT-MAP.md   # 多上下文时,指向各自的 CONTEXT.md
├── docs/
│   ├── adr/0001-xxx.md    # 架构决策(四位数编号,极简)
│   └── modules/AUTH.md    # 只放复杂模块(按需)
└── .claude/skills/        # 6 个技能
```
