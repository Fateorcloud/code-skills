# 混合方案启动模板

将以下文件复制到你的新项目：

---

## 📄 CLAUDE.md

```markdown
# [项目名称]

**项目简介：** [一句话描述]

---

## 🤖 AI 协作指南

### 文档体系

本项目结合两种文档方法：

1. **CONTEXT.md** - 领域语言字典
   - 定义项目特有的术语
   - 保持精简，只记录核心概念
   - 遇到不熟悉的术语，先查这里

2. **docs/adr/** - 架构决策记录
   - 记录"为什么这样设计"
   - 重大技术选型的原因
   - 按时间顺序编号（001, 002...）

3. **docs/modules/** - 复杂模块规范（按需创建）
   - 对安全敏感或复杂的模块提供详细 spec
   - 如：认证、支付、数据导出
   - 包含接口定义、流程图、约束条件

### 工作流程

#### 快速开发（使用 CONTEXT.md）

```
用户："使用 CONTEXT.md 的术语，实现课程列表功能"
AI：读取术语 → 使用一致的命名 → 快速实现
```

#### 严格规范（使用模块文档）

```
用户："严格按照 docs/modules/AUTH.md 实现用户登录"
AI：读取详细规范 → 按 spec 执行 → 确保安全性
```

#### 增量更新（使用技能）

```
用户："使用 /grill-with-docs 来理解这个新功能"
AI：主动提问 → 理解需求 → 更新 CONTEXT.md
```

## 💡 何时创建详细模块文档？

当模块满足以下任一条件时，创建 `docs/modules/XXX.md`：

- ✅ 涉及安全（认证、授权、加密）
- ✅ 处理支付或敏感数据
- ✅ 业务逻辑复杂（多步骤流程）
- ✅ 多人协作需要统一理解
- ✅ AI 反复理解错误

其他模块保持灵活，只在 CONTEXT.md 中定义术语即可。

---
```

## 📄 CONTEXT.md

```markdown
# 项目领域语言

**目的：** 定义项目特有的术语，让 AI 使用一致的命名

---

## 核心概念

### [领域1]

- **[术语1]** - [简短定义]
- **[术语2]** - [简短定义]

### [领域2]

- **[术语3]** - [简短定义]

---

## 示例

### 电商项目

- **Product** - 商品实体
- **SKU** - 商品库存单位（Stock Keeping Unit）
- **Cart** - 购物车，存储在 Redis
- **Order** - 订单，支付后从 Cart 生成
- **Fulfillment** - 订单履约流程（打包、发货、签收）

### 教育平台

- **Course** - 课程
- **Section** - 章节
- **Lesson** - 小节  
- **Materialization** - 课程发布流程（draft → review → published）
- **Enrollment** - 学员报名关系

---

**维护规则：**
- 新术语出现时立即添加
- 保持定义简短（1-2句话）
- 避免技术实现细节（那属于模块文档）
- 使用 /grill-with-docs 自动更新
```
