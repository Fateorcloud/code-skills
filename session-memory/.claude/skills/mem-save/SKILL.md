# mem-save — 把当前对话压成一份可接手的归档

> 目的：多进程开发时各对话上下文不互通、越聊越贵。把当前对话压成**高密度归档**存进项目，新进程用 `/mem-load` 接手，不必重放整段历史。
> 关键：直接从**你当前的上下文**总结落盘，不用去翻 transcript。压缩要的是信息密度，不是流水账。

## 步骤

### 1. 定 slug（归档文件名）
- 优先用**当前 git 分支名** → slug；或用户给的主题；都没有就按任务主题起一个短横线 slug（如 `refactor-auth`）。
- 查 `.agent-memory/<slug>.md` 是否已存在 → **存在就更新合并**（把新进展并进去、`updated` 改今天），**不新建重复文件**。

### 2. 首次懒创建目录 + 关 git
`.agent-memory/` 不存在则创建，并写 `.agent-memory/.gitignore`：
```
# 进程对话记忆：本地私有，不入库
*
!.gitignore
```

### 3. 写归档 `.agent-memory/<slug>.md`
按下面模板，**只写有信息量的段**（空段直接跳过）：
```md
---
slug: <slug>
branch: <git 分支/无>
status: active | blocked | done
updated: <YYYY-MM-DD>
---

# <一句话：这对话在干什么>

## 目标
<要解决的问题/功能，1-3 句>

## 关键决策
- <决策 → 为什么>

## 现状
- 已完成 / 进行中 / 已验证可用

## 改动文件
- `path:line` — 做了什么

## 下一步
1. …

## 坑 / 待解决
- <阻塞、踩坑、未决>

## 导航指针（让新对话别重读全部）
- 入口：`path`   相关：`path`/关键函数
```

抓取要点：**目标、关键决策（带为什么）、现状、改动文件（带 `path:line`）、下一步、坑、导航指针**。

### 4. 更新索引 `.agent-memory/INDEX.md`
该 slug 已有行就**改那行**，没有就**追加一行**；索引不存在则先建带标题的空索引：
```md
# 进程对话记忆索引
> 一行一条。新对话用 /mem-load 先读本文件，再按需加载具体归档。

- [<slug>](<slug>.md) — <一句话> · `status` · `<branch>` · <updated>
```

### 5. 报告 + 指下一步
告诉用户：存了哪个文件 + 一句话摘要。提示：可以 `/clear` 或另开一个 `claude` 进程，再用 `/mem-load` 接手，省 token。

## 原则
- **密度优先**：抓决策、现状、下一步、坑、导航指针；别抄对话原文。
- **一对话一归档**：同一任务反复存 → 更新同一个 slug，别刷屏建新文件。
- 决策**难逆 + 反直觉 + 真权衡** → 提示走 `/adr`（若项目装了 Spec-skill）。
- 归档是**本地私有**（已 gitignore），别提交进库。
