# mem-load — 新对话读归档、接手上一段工作

> 目的：让一个全新（省 token）的对话，快速拿到其它进程 `/mem-save` 存下的进度，不必重讲一遍。
> 关键：**按需加载**——先读轻量索引，再只读相关归档；别把所有归档全灌进来，否则又把 token 顶上去，违背初衷。

## 步骤

### 1. 读索引
读 `.agent-memory/INDEX.md`。
- 不存在或为空 → 告诉用户「本项目还没有归档」，正常开工。

### 2. 挑相关归档
结合用户这次要做什么 + 当前 git 分支，从索引里挑最相关的 1~N 条：

| 情况 | 怎么做 |
|------|--------|
| 用户指名（「读 `<slug>`」「读全部」） | 照办 |
| 只有一条明显相关 | 直接读它 |
| 多条都相关、又没指名 | **列出候选让用户选，别默认全读** |
| 没有相关的 | 说明情况，正常开工 |

### 3. 只读选中的归档
`Read` 选中的 `.agent-memory/<slug>.md`，把 **目标 / 现状 / 下一步 / 坑 / 导航指针** 载入上下文。别把没选中的也读进来。

### 4. 出接手简报
给用户一段简报：**上次做到哪、下一步是什么、有哪些坑、关键文件在哪**；然后按「下一步」继续干。

## 原则
- **省 token 是第一目的**：只读需要的，读完直接干，别复述整份归档。
- **归档可能过时**：导航指针指到的文件，动手前先看一眼现状对不对。
- 接着做出了新进展 → 用 `/mem-save` 更新**同一个 slug**，保持接力链不断。
